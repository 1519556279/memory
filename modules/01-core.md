# Module 01: Core — 核心概念与初始化

## 什么是记忆系统

记忆系统是一个持久化存储层，让 AI 在跨会话时保持对你的了解。
它通过文件系统存储三类信息：

| 信息类型 | 文件 | 说明 |
|---------|------|------|
| 偏好 (Preferences) | preferences.md | 你喜欢/不喜欢什么，你的习惯风格 |
| 事实 (Facts) | facts.md | 已做的决定、项目状态、重要事件 |
| 存档 (Archive) | archive.jsonl | 压缩后的旧对话摘要，用于省 Tokens |

## 初始化流程

每次新对话开始，自动按顺序执行以下步骤：

### Step 1: 检测记忆目录

```
检查路径：~/.reasonix/memory/projects/<project-id>/
```

`<project-id>` 根据当前项目自动识别。例如 D:/Reasonix 对应 `reasonix`。

### Step 2: 加载记忆文件

如果目录存在，按顺序加载：

1. `preferences.md` → 读取用户偏好（见 Module 02）
2. `facts.md` → 读取关键事实（见 Module 02）
3. `index.json` → 读取索引（见 Module 03）
4. `archive.jsonl` → 读取最近存档摘要（见 Module 04）

### Step 3: 输出恢复报告

```
🧠 记忆已恢复
━━━━━━━━━━━━━━━━━━
📋 偏好记录: N 条
📌 关键事实: N 条
📦 压缩存档: N 条
━━━━━━━━━━━━━━━━━━
```

### Step 4: 注入上下文

将偏好和事实注入到当前对话上下文，格式见 Module 05。

## 生命周期

```
┌─────────────────────────────────────┐
│         新对话开始                    │
│     ↓ 自动检测记忆文件               │
│     ↓ 加载并输出恢复报告             │
│     ↓ 注入到上下文                   │
├─────────────────────────────────────┤
│         对话进行中                    │
│     ↓ 自动保存偏好/事实              │
│     ↓ 对话超阈值时压缩              │
│     ↓ 用户命令时响应                │
├─────────────────────────────────────┤
│         对话结束                     │
│     （无需特殊操作，已实时写入）      │
├─────────────────────────────────────┤
│         下次新对话                   │
│     ↓ 回到第一步，自动恢复           │
└─────────────────────────────────────┘
```

## 跨会话工作流

```
对话 A
  └── 你说：我喜欢用 TypeScript
      └── 写入 preferences.md
  └── 对话结束

对话 B（新对话）
  └── 读取 preferences.md
  └── 恢复：📋 偏好记录: 1 条
  └── 你不用说"我用 TypeScript"，我已经知道了
```

## 核心原则

1. **文件即记忆**：所有数据存在硬盘，不依赖会话上下文
2. **按项目隔离**：不同项目的记忆互不干扰
3. **自动为主**：99% 的操作无需手动干预
4. **增量保存**：每次只更新变化的部分，不重写整个文件
5. **实时写入**：检测到偏好/事实立即保存，不怕对话中断
