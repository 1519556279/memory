# Module 03: Index — 索引与检索

## 索引的作用

没有索引 → 每次搜索需要扫描整个文件 → 慢、不准

有索引 → 先查 index.json 找到位置 → 直接定位 → 快、准

## 索引数据结构

每条记忆在 index.json 中记录：

```json
{
  "id": "pref-001",
  "content": "TypeScript，2空格缩进",
  "tags": ["代码", "偏好"],
  "date": "2024-06-07",
  "keywords": ["typescript", "ts", "2空格", "缩进"]
}
```

## 关键词提取规则

自动从保存内容中提取关键词：

| 原文 | 提取的关键词 |
|------|------------|
| "我喜欢用 TypeScript" | typescript, ts |
| "代码用 2 空格缩进" | 2空格, 缩进 |
| "删除了 computer-control" | 删除, computer-control |
| "仙侠小说，主角林峰" | 仙侠, 小说, 林峰 |
| "用 pnpm 代替 npm" | pnpm, npm |

提取规则：
- 去掉停用词（的、了、是、在、有、我、你、他）
- 提取名词和关键动词
- 英文词保留原样 + 小写版
- 中文词保留原样

## 标签系统

每条记忆自动分配 1-3 个标签：

| 标签 | 自动分配规则 |
|------|------------|
| 代码 | 内容包含：TypeScript/React/API/组件/函数/类 |
| 小说 | 内容包含：小说/角色/章节/剧情/世界观 |
| 偏好 | 内容包含：喜欢/讨厌/习惯/风格/用 |
| 决定 | 内容包含：删了/装了/决定/重构/改 |
| 项目 | 内容包含：进度/安装了/上传/部署 |
| 设计 | 内容包含：配色/设计/布局/UI |

## 搜索算法

### 关键词搜索（精确匹配）

```
输入："pnpm"
→ 在 index.json 的 keywords 数组中逐条匹配
→ 返回 keywords 包含 "pnpm" 的所有条目
→ 按日期倒序排列
→ 取 Top-5
```

### 标签过滤（精确筛选）

```
输入：#代码
→ 在 index.json 中过滤 tags 包含 "代码" 的条目
→ 按日期倒序排列
```

### 全文搜索（模糊匹配）

```
输入："我之前说过喜欢用 pnpm"
→ 提取关键词：喜欢、pnpm
→ 在 index.json 搜索 keywords 包含它们
→ 同时在 preferences.md 全文搜索
→ 同时在 facts.md 全文搜索
→ 合并去重，按相关度排序
```

### 相关度排序规则

| 优先级 | 匹配类型 | 得分 |
|-------|---------|------|
| 1 | 标签精确匹配 | 100 |
| 2 | 关键词完全匹配 | 80 |
| 3 | 关键词部分匹配 | 60 |
| 4 | 全文匹配 | 40 |
| 5 | 日期倒序（新优先） | +10 |

## 索引维护

| 操作 | 触发时机 |
|------|---------|
| 新增条目 | 保存偏好/事实时 |
| 删除条目 | /忘记 命令时 |
| 重建索引 | index.json 损坏或 /清理记忆 时 |
| 压缩索引 | 超过 500KB 时 |
