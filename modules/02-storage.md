# Module 02: Storage — 记忆存储格式

## 文件总览

| 文件 | 格式 | 用途 | 自动维护 |
|------|------|------|---------|
| preferences.md | Markdown | 用户偏好/习惯/风格 | ✅ |
| facts.md | Markdown | 关键事实/决策/状态 | ✅ |
| index.json | JSON | 标签+关键词索引 | ✅ |
| archive.jsonl | JSONL | 压缩对话摘要 | ✅ |

---

## 1. preferences.md — 偏好记录

记录用户的喜好、习惯、风格要求。按日期分组，最新的在最前面。

### 格式规范

```markdown
# 偏好记录

## 2024-06-07
- [标签]：[内容]

## 2024-06-06
- [标签]：[内容]
```

### 实际示例

```markdown
# 偏好记录

## 2024-06-07
- 编程：TypeScript，2 空格缩进，禁止 any
- 写作：仙侠小说，主角名林峰
- 设计：Apple 简洁风格

## 2024-06-06
- 工具：用 pnpm 代替 npm
- 代码：命名用 camelCase
```

### 自动保存规则

| 触发场景 | 用户说的话 | 保存内容 |
|---------|-----------|---------|
| 明确喜好 | "我喜欢用 TypeScript" | 编程：TypeScript |
| 明确反感 | "我不喜欢 any 类型" | 编程：禁止 any |
| 提出要求 | "代码用 2 空格缩进" | 编程：2 空格 |
| 表达习惯 | "我习惯先写测试" | 工作流：TDD |
| 表达风格 | "我喜欢简洁风格" | 设计：简洁风格 |

## 2. facts.md — 关键事实

记录已做的决策、项目进度、重要事件。

### 格式规范

```markdown
# 关键事实

## 2024-06-07
- [标签]：[内容]

## 2024-06-06
- [标签]：[内容]
```

### 实际示例

```markdown
# 关键事实

## 2024-06-07
- 项目：安装了 tianming-skill（小说创作）
- 删除：删除了 computer-control
- 进度：记忆系统升级为多模块结构

## 2024-06-06
- 安装：上传到 GitHub
- 决定：合并到 main 主线
```

### 自动保存规则

| 触发场景 | 用户说的话 | 保存内容 |
|---------|-----------|---------|
| 做决定 | "把这个功能删掉" | 删除了 xxx |
| 报进度 | "写到第 5 章了" | 小说进度：第 5 章 |
| 安装/卸载 | "帮我装个 xxx" | 安装了 xxx |
| 重要事件 | "我今天重构了" | 重构了 xxx |

## 3. index.json — 索引文件

用于快速检索的关键词索引。

### 格式规范

```json
{
  "preferences": [
    {
      "id": "pref-001",
      "content": "喜好摘要",
      "tags": ["标签"],
      "date": "2024-06-07",
      "keywords": ["关键词1", "关键词2"]
    }
  ],
  "facts": [
    {
      "id": "fact-001",
      "content": "事实摘要",
      "tags": ["标签"],
      "date": "2024-06-07",
      "keywords": ["关键词1", "关键词2"]
    }
  ],
  "lastUpdated": "2024-06-07"
}
```

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| id | 唯一标识 | pref-001, fact-001 |
| content | 摘要内容 | TypeScript |
| tags | 主题标签 | ["代码"] |
| date | 记录日期 | 2024-06-07 |
| keywords | 检索关键词 | ["typescript", "ts"] |

### 标签系统

| 标签 | 含义 | 适用场景 |
|------|------|---------|
| 代码 | 编程相关 | TypeScript, React, API |
| 小说 | 写作相关 | 角色, 世界观, 情节 |
| 偏好 | 用户喜好 | 风格, 习惯, 工具 |
| 决定 | 关键决策 | 安装, 删除, 重构 |
| 项目 | 项目状态 | 进度, 里程碑, 版本 |
| 设计 | UI/UX | 配色, 布局, 风格 |

## 4. archive.jsonl — 压缩存档

每行一条 JSON 对象，记录被压缩的旧对话摘要。

### 格式规范

```jsonl
{"date":"2024-06-07","summary":"对话摘要","tags":["标签"],"tokens_saved":2000,"importance":3}
{"date":"2024-06-06","summary":"对话摘要","tags":["标签"],"tokens_saved":3500,"importance":4}
```

### 字段说明

| 字段 | 说明 | 范围 |
|------|------|------|
| date | 压缩日期 | YYYY-MM-DD |
| summary | 摘要文本 | 1-3 句话 |
| tags | 主题标签 | 1-3 个 |
| tokens_saved | 节省 Token 数 | 数值 |
| importance | 重要性 | 1(低) ~ 5(高) |

---

## 文件大小控制

| 文件 | 大小阈值 | 超限处理 |
|------|---------|---------|
| preferences.md | 100 KB | 自动归档最早的 50% |
| facts.md | 100 KB | 自动归档最早的 50% |
| index.json | 500 KB | 重建索引，移除低价值条目 |
| archive.jsonl | 1 MB | 清理重要性 < 3 的条目 |
