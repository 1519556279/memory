# Module 05: Retrieval — 记忆检索与注入

## 检索触发词

当用户消息包含以下关键词时，自动触发记忆检索：

| 触发词 | 示例 | 检索范围 |
|--------|------|---------|
| 记得 | "记得我之前说过..." | 全部 |
| 之前 | "我之前是不是说了..." | 全部 |
| 说过 | "我说过用 pnpm 吗" | 全部 |
| 偏好 | "我的偏好是什么" | preferences.md |
| 习惯 | "我有什么习惯" | preferences.md |
| 喜欢 | "我喜欢什么风格" | preferences.md |
| 决定 | "我们做了什么决定" | facts.md |
| 忘记 | "我忘记了..." | index.json |
| /记忆 | 手动命令 | 全部 |
| 记住 | "帮我记住..." | 存 → 不搜 |

## 检索流程

```
收到用户消息
    ↓
Step 1: 匹配触发词
    ↓ 不匹配 → 无操作，正常对话
    ↓ 匹配 → 进入 Step 2
    ↓
Step 2: 确定检索范围
    → "记得/之前/说过" → 全部文件
    → "偏好/喜欢/习惯" → preferences.md
    → "决定/进度/状态" → facts.md
    → "/记忆" → 全部文件
    ↓
Step 3: 执行搜索
    → 提取关键词
    → 在 index.json 中搜索 keywords 匹配
    → 在对应文件中全文搜索
    ↓
Step 4: 排序结果
    → 按相关度得分（见 Module 03）
    → 取 Top-5
    ↓
Step 5: 格式化输出
    → 整理成易读格式
    → 注入到上下文
```

## 注入格式

检索到的记忆以以下格式注入到对话上下文：

```
[记忆上下文]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 偏好记录：
  · TypeScript，2 空格缩进（#代码）
  · 仙侠小说，主角林峰（#小说）
  · Apple 简洁设计风格（#设计）

📌 关键事实：
  · 删除了 computer-control（#决定）
  · 安装了 tianming-skill（#项目）
  · 当前小说进度：第 3 章（#项目）

📦 最近存档摘要：
  · 重构了记忆系统（2024-06-07）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 按场景注入

| 用户场景 | 注入重点 |
|---------|---------|
| 写代码 | 代码相关偏好（语言、风格、工具） |
| 写小说 | 小说相关偏好（题材、风格、进度） |
| 设计页面 | 设计相关偏好（配色、风格） |
| 日常对话 | 通用偏好 + 最近事实 |
| 新对话开始 | 全部偏好 + 最新事实 + 最近存档 |

## 注入时机

| 时机 | 操作 |
|------|------|
| 新对话开始 | 自动注入全部记忆 |
| 用户切换话题 | 检测到话题变化时重新注入 |
| 用户明确问"记得..." | 触发检索并注入匹配结果 |
| 用户说"我的..." | 触发偏好检索 |

## 检索失败处理

| 情况 | 处理方式 |
|------|---------|
| 没有匹配结果 | 回复："目前没有找到相关记忆，你可以说 /记住 来保存" |
| index.json 不存在 | 降级为全文搜索 preferences.md + facts.md |
| 文件损坏 | 回复："记忆文件似乎有问题，试试 /清理记忆" |
| 首次使用（无记忆文件） | 回复："还没有记忆记录，从现在开始我会记住的" |
