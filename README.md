## 🎯 解决的问题

**AI 没有持久记忆。** 每次新对话都是全新的，导致：

| 问题 | 后果 | 本技能如何解决 |
|------|------|--------------|
| 你要反复说同样的需求和偏好 | 浪费 Tokens，重复劳动 | 一次记录，永久记住 |
| 写到一半关掉，下次就忘了 | 创作中断，需要从头回忆 | 跨会话自动恢复 |
| 对话长了最早的内容被丢弃 | 早期设定/偏好丢失 | 自动压缩旧对话 |
| 不记得之前做过什么决定 | 重复决策，前后矛盾 | 关键词索引，一搜就到 |

**核心原理：文件即记忆。所有数据存在硬盘，不依赖会话上下文。**

## ✨ 功能特性

### 通用记录，不分领域

不再区分代码/小说/日常对话。所有重要信息都会被记录：

- **偏好**：你喜欢 TypeScript / 2 空格缩进 / 仙侠小说 / 简洁设计
- **事实**：你删除了某个功能 / 做了某个决定 / 项目进度
- **上下文**：你在做什么、你的目标是什么

### 自动压缩上下文，节约 Tokens

对话超过一定轮次后，最早的对话自动压缩为摘要，保留关键信息扔掉冗余内容。

```
传统方式：从头聊 → 我啥都不记得 → 你重复说 → 浪费 Tokens
记忆方式：恢复记忆 → 直接继续 → 省 Tokens
```

### 跨对话恢复

今天聊完关掉，明天新对话自动加载记忆，你不需要重新说一遍。

### 关键词索引检索

每条记忆自动标注标签和关键词。说「记得我上次说过...」就能搜到。

### 按项目隔离

A 项目的记忆不会混到 B 项目。

## 🚀 快速开始

### 安装

```bash
# 直接克隆
git clone https://github.com/1519556279/memory.git

# 放到你的 AI agent 可访问的路径
```

### 第一次使用

直接说「**帮我记住 XXX**」或「**/记住 XXX**」开始建立记忆。

### 新对话恢复

每次新对话开始，自动检测并输出恢复报告：

```
🧠 记忆已恢复
━━━━━━━━━━━━━━━━━━
📋 偏好记录: 3 条
📌 关键事实: 4 条
📦 压缩存档: 2 条
━━━━━━━━━━━━━━━━━━
```

## 🎮 手动命令

| 命令 | 作用 | 示例 |
|------|------|------|
| `/记住 内容` | 手动保存偏好或事实 | `/记住 我喜欢用 pnpm` |
| `/忘记 关键词` | 删除记忆 | `/忘记 pnpm` |
| `/记忆` | 查看所有记忆 | `/记忆` |
| `/记忆 #标签` | 按标签过滤 | `/记忆 #代码` |
| `/压缩记忆` | 手动压缩对话 | `/压缩记忆` |
| `/清理记忆` | 整理低价值记忆 | `/清理记忆` |

### 实际使用效果

```
用户：/记住 我喜欢用 TypeScript 和 2 空格缩进
→ ✅ 已记住：编程偏好 — TypeScript，2 空格缩进

用户：/记忆
→ 📋 偏好: 3 条 | 📌 事实: 4 条 | 📦 存档: 2 条

用户：我之前是不是说过喜欢用 pnpm？
→ 🔍 找到匹配记忆：用 pnpm 代替 npm（2024-06-07 记录）
```

## 🔄 工作流程

```
新对话开始
    ↓
加载 01-core → 检测记忆文件
    ↓
有记忆？ → 加载 02-storage → 读取 → 输出恢复报告
    ↓
用户说话
    ↓
匹配"记得/之前/说过"？ → 加载 03-index + 05-retrieval → 搜索 → 注入
    ↓
匹配偏好/事实？ → 加载 02-storage → 保存 → 更新索引
    ↓
超过 50 轮？ → 加载 04-compression → 压缩 → 存档
    ↓
用户输命令？ → 加载 06-commands → 执行
```

## 🛠 技术栈

- **格式**: Markdown + JSON + JSONL
- **存储**: 本地文件系统（按项目隔离）
- **编码**: UTF-8
- **索引**: 关键词匹配 + 标签系统 + 全文搜索
- **压缩**: 增量式对话摘要
